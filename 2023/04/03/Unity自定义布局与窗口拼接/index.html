Unity自定义布局与窗口拼接

之前在做Unity窗口的时候，因为使用的是原生的IMGUI，所以对于单个窗口中可能出现的多个模块，可以进行自定义的布局，即可以自己控制每一个模块的显示区域，然后单独绘制。然而在引入Odin进行开发之后，每个模块都被单独拆分出成一个窗口，虽然Odin支持自定义绘制，但是在开发的时候又觉得这样过于麻烦。而Unity本身不支持窗口的自定义拼接，即多个窗口打开时是散乱排列的，做不到像Unity编辑器窗口这样有一个自动的布局，在网上找了一下之后，看到大佬提供了一个通过反射的思路拿到Unity内置的接口，去做自定义布局的方案，尝试之后发现确实能够实现我需要的功能，这里做一个简单的记载。
Untiy本身对窗口或者说对于窗口所属的容器分为了以下四种：

  1. ContainerWindow（最外层容器，根容器节点）
  2. SplitView（窗口分割容器，可以嵌套在ContainerWindow之中，也可以嵌套自身SplitView类型的容器）
  3. DockArea（作为EditorWindow的父节点容器存在）
  4. EditorWindow（自定义界面）

即最终的窗口结构为：ContainerWindow -> SplitView -> SplitView/DockArea -> EditorWindow
